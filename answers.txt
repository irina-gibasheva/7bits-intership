1. Согласно профайлеру, временные затраты связаны с тем, что запросы на сервер происходят довольно долго. Время, уходящее
на выполнение операций на стороне скрипта гораздо меньше времени, уходящего на коммуникацию с сервером.

2. Чтобы уменьшить время выполнения алгоритма, следует уменьшить количество запросов на сервер. В связи с этим хотелось
бы добавить запросы для проверки состояния вагона по номеру от текущего вперёд и назад и запросы для перехода вперёд и
 назад на заданное количество вагонов. Например такие:
GET :key/train/next/state/n перейти в n-й вагон от текущего вперёд и проверить, включён ли свет {success: true, state: true/false}
GET :key/train/previous/state/n перейти в n-й вагон от текущего назад и проверить, включён ли свет {success: true, state: true/false}
POST :key/train/next/n идем в вагон под номером n от текущего вперёд, ответ {success: true}
POST :key/train/previous/n идем в вагон под номером n от текущего назад, ответ {success: true}

3. Импользование опции async: false означает, что запрос не будет выполняться в асинхронном режиме. Это значит, что запрос
должен быть выполнен до того, как начнёт выполняться следущий запрос. Это сделано для того, чтобы до того, как алгоритм
выполнится, пользователь не мог нажимать кнопки и отправлять таким образом другие запросы, которые могут помешать
исполнению алгоритма.

4. Обычно XMLHttpRequest может делать запрос только в рамках текущего сайта. При попытке использовать другой домен/порт/протокол –
браузер выдаёт ошибку. Опция withCredentials: true позволяет отправлять кросс-доменные запросы, то есть отправлять запросы
на сервер 7bits: http://frontend-internship.7bits.it/, реализующий API общения с поездом.
